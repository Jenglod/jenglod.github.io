Pendahuluan

Mikrotik adalah sistem operasi dan perangkat keras yang dirancang untuk fungsi router dan firewall. Produk ini dapat digunakan untuk berbagai kebutuhan jaringan, dari jaringan kecil di rumah atau kantor hingga jaringan ISP (Internet Service Provider) skala besar. Mikrotik menyediakan solusi perangkat lunak (Mikrotik RouterOS) dan perangkat keras (RouterBoard) yang dapat dikonfigurasi untuk melakukan berbagai fungsi jaringan seperti routing, firewall, bandwidth management, VPN, dan hotspot.

Fungsi Utama Mikrotik

  • Routing: Mengatur lalu lintas data antar jaringan.
  • Firewall: Melindungi jaringan dari akses tidak sah dan ancaman keamanan.
  • Bandwidth Management: Mengelola dan mengalokasikan penggunaan bandwidth pada jaringan.
  • VPN (Virtual Private Network): Mengamankan koneksi jaringan melalui tunneling.
  • Hotspot: Mengelola jaringan wireless untuk koneksi publik dengan kontrol akses.
  • QoS (Quality of Service): Mengatur prioritas trafik jaringan.

Tipe Lab Mikrotik

Basic Routing

Lab ini digunakan untuk mempelajari dasar-dasar routing, pengenalan protokol routing, dan cara mengonfigurasi jaringan LAN atau WAN. Peserta belajar bagaimana cara membuat jaringan terhubung dan saling berkomunikasi melalui Mikrotik.

Topik yang dipelajari: IP addressing, static routing, dan dynamic routing (RIP, OSPF).

Contoh Pengaturan: Mengonfigurasi static route untuk menghubungkan dua jaringan.

Bandwidth Management

Lab ini berfokus pada pengaturan alokasi bandwidth penting untuk jaringan dengan sumber daya terbatas atau ketika banyak pengguna menggunakan internet secara bersamaan.

Topik yang dipelajari: Queue, burst, simple queue, dan queue tree.

Contoh Pengaturan: Membatasi bandwidth untuk pengguna tertentu, misalnya 2 Mbps untuk download dan 512 Kbps untuk upload.

Firewall dan Keamanan Jaringan

Firewall digunakan untuk melindungi jaringan dari serangan dan akses tidak sah. Dalam lab ini, peserta belajar mengonfigurasi filter rules, NAT (Network Address Translation), dan mangle untuk meningkatkan keamanan jaringan.

Topik yang dipelajari: Chain (input, forward, output), filter rules, NAT, dan address list.

Contoh Pengaturan: Blokir akses dari IP atau port tertentu untuk melindungi jaringan dari serangan.

Hotspot

Lab ini digunakan untuk membangun dan mengelola jaringan hotspot. Dalam sesi ini, peserta belajar bagaimana cara mengonfigurasi Mikrotik sebagai server hotspot untuk menyediakan akses internet publik yang dapat dikontrol.

Topik yang dipelajari: Hotspot, user profiles, walled garden, bandwidth limitation, dan login pages.

Contoh Pengaturan: Mengonfigurasi halaman login untuk pengguna yang ingin mengakses internet melalui jaringan hotspot.

VPN Configuration

VPN memungkinkan pengguna untuk mengakses jaringan lokal dari jarak jauh melalui internet dengan aman. Lab VPN Mikrotik berfokus pada cara membangun VPN menggunakan berbagai protokol seperti PPTP, L2TP, atau OpenVPN.

Topik yang dipelajari: PPP (Point-to-Point Protocol), PPTP, L2TP, IPSec.

Contoh Pengaturan: Membuat tunnel VPN untuk menghubungkan dua jaringan yang berbeda secara geografis.

Konfigurasi Mikrotik

CLI (Command Line Interface)

CLI adalah salah satu cara yang paling umum digunakan oleh pengguna yang berpengalaman. Dengan CLI, pengguna bisa memberikan perintah langsung melalui terminal untuk mengonfigurasi router.

Contoh Perintah CLI:

/ip address add address=192.168.88.1/24 interface=ether1
/ip route add dst-address=0.0.0.0/0 gateway=192.168.88.254

WebFig

WebFig adalah antarmuka berbasis web yang memungkinkan pengguna mengakses Mikrotik RouterOS melalui browser. Dengan WebFig, pengguna dapat melakukan konfigurasi router tanpa perlu menginstal aplikasi tambahan.

Winbox

Winbox adalah aplikasi grafis untuk mengelola dan mengonfigurasi router Mikrotik secara mudah. Winbox menawarkan antarmuka yang lebih ramah pengguna dibandingkan CLI dan memungkinkan pengelolaan router dengan cara yang lebih interaktif.

Langkah-langkah Menggunakan Winbox:

  • Unduh dan Instal: Unduh aplikasi Winbox dari situs resmi Mikrotik dan jalankan.
  • Koneksi ke Router: Router yang ada di jaringan akan terdeteksi secara otomatis pada daftar. Masukkan username dan password.
  • Konfigurasi IP Address: Masuk ke menu IP > Addresses. Tambahkan IP address dan interface yang digunakan, misalnya 192.168.88.1/24 pada ether1.
  • Konfigurasi DHCP: Masuk ke IP > DHCP Server untuk membuat DHCP server baru dengan interface yang digunakan dan rentang IP address untuk klien.
  • Konfigurasi NAT: Tambahkan aturan srcnat pada IP > Firewall > NAT untuk memungkinkan NAT mengalirkan trafik dari jaringan internal ke internet.
  • Monitoring: Gunakan tab Interfaces atau Resources untuk melihat status interface, trafik jaringan, dan penggunaan sumber daya router secara real-time.
